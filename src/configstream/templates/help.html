Frequently Asked Questions (FAQ)

Why does the script take so long to run?

The `merge` command tests hundreds of servers for connectivity and latency. You can speed this up by reducing the number of sources in `sources.txt` or using a smaller `--concurrent-limit`.

Where are my output files?

By default, all generated files are saved in the `output/` directory in the project root. You can change this with the `--output-dir` argument.

How do I use the generated subscription link?

Copy the contents of `output/vpn_subscription_base64.txt` and import it into a compatible client like Hiddify, v2rayNG, or Clash.

Core Concepts

Fetching vs. Merging

Fetching (`configstream fetch`) gathers raw configuration links from sources without testing them. Merging (`configstream merge`) tests those configurations, filters out non-working ones, sorts them by performance, and generates the final subscription files.

Configuration (`config.yaml`)

This file is the heart of your setup. You can define which protocols to fetch, set network timeouts, configure API keys, and much more. Always back up your `config.yaml` file.
