Quick Start Guide

Get up and running with ConfigStream in just a few simple steps.

1. Installation

First, ensure you have Python 3.8+ installed. Then, install ConfigStream and its dependencies by cloning the repository and using pip:

git clone https://github.com/amirrezafarnamtaheri/configStream.git
cd configStream
pip install .

For development, which includes tools for testing and linting, install the [dev] extras:

pip install -e .[dev]

2. Configuration

Copy the example configuration file to create your own:

cp config.yaml.example config.yaml

Next, create a sources.txt file and add your subscription URLs, one per line. This file tells ConfigStream where to fetch proxy configurations from.

https://raw.githubusercontent.com/barry-far/V2ray-Configs/main/All_Configs_Sub.txt
https://raw.githubusercontent.com/yebekhe/V2Hub/main/merged

3. Running the Daemon

The simplest way to start is by running the daemon, which launches the web dashboard and a background scheduler for automatic testing:

configstream daemon

You can then access the web interface at http://localhost:8080.

4. Command-Line Usage

For more direct control, you can use the command-line interface to fetch, test, and merge configurations:

  • Fetch: configstream fetch - Gathers configurations from your sources.
  • Test: configstream retest - Tests the fetched configurations for latency and reliability.
  • Merge: configstream merge - Combines the tested configurations into a single subscription file.
  • Full Cycle: configstream full - Runs the entire fetch, test, and merge process in one command.

5. Exploring the Dashboard

The dashboard provides a comprehensive overview of your proxy configurations. You can monitor their status, view performance metrics, and export them in various formats. It's the best way to visualize and manage your proxy collection.
