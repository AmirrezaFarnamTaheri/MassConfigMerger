Project Roadmap

Our vision for the future of MassConfigMerger. This is a living document and may change based on community feedback.

Phase 1: Core Functionality Completed

  • CLI for fetching, merging, and testing
  • Support for multiple protocols (VMess, VLESS, etc.)
  • Output generation for Clash and Sing-Box
  • Basic filtering (protocol, country)
  • Docker support

Phase 2: Enhancements & Reliability In Progress

  • Advanced testing (bandwidth, packet loss, jitter)
  • Enhanced security checks (IP reputation, certificate validation)
  • Historical performance tracking and reliability sorting
  • Web interface for monitoring and control
  • Improved documentation and GitHub Pages site

Phase 3: Extensibility & Usability Planned

  • Plugin system for custom parsers and output formats
  • Interactive TUI (Text-based User Interface)
  • REST API for programmatic control
  • Advanced filtering with complex expressions
  • Support for more VPN clients and protocols

Phase 4: Ecosystem Integration Planned

  • Prometheus exporter for metrics
  • Kubernetes deployment manifests (Helm chart)
  • Integration with notification services (Pushover, etc.)
  • Official Python SDK for developers
