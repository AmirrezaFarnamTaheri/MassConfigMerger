Quick Start Guide

Get up and running with MassConfigMerger in 5 minutes

Installation

Requirements: Python 3.8 or higher

Method 1: Using pip (Recommended)

# Install from PyPI
pip install massconfigmerger

# Verify installation
massconfigmerger --version

Method 2: Using Docker

# Pull the image
docker pull massconfigmerger/massconfigmerger:latest

# Run a command
docker run -v $(pwd)/output:/app/output \
    massconfigmerger/massconfigmerger \
    massconfigmerger full

Method 3: From Source

# Clone the repository
git clone https://github.com/yourusername/MassConfigMerger.git
cd MassConfigMerger

# Install in development mode
pip install -e .

# Or install with all extras
pip install -e .[dev,web]

First Run

Let's run your first aggregation and merge:

1

Run the Full Pipeline

This command will fetch, test, and merge configurations:

massconfigmerger full

This may take 5-15 minutes depending on your internet connection

2

Check the Output

Your subscription files will be in the output/ directory:

ls output/

# You should see:
# vpn_subscription_raw.txt       - Raw configuration list
# vpn_subscription_base64.txt    - Base64 encoded (for import)
# clash.yaml                     - Clash configuration
# vpn_singbox.json              - Sing-Box configuration
# vpn_report.json               - Detailed report
# vpn_detailed.csv              - CSV with all details
3

Import to Your VPN Client

Use the appropriate file for your VPN client:

  • Clash/Meta: Use clash.yaml
  • Sing-Box/Stash: Use vpn_singbox.json
  • V2RayN/V2RayNG: Import vpn_subscription_base64.txt
  • Other clients: Use vpn_subscription_raw.txt
Congratulations! You've successfully generated your first VPN subscription.

Configuration

Create a config.yaml file in your project directory to customize settings:

network:
  concurrent_limit: 20        # Number of concurrent downloads
  request_timeout: 10         # Timeout in seconds

filtering:
  include_countries: [US, GB, JP]  # Only these countries
  fetch_protocols: [vmess, vless, trojan]  # Only these protocols

processing:
  enable_url_testing: true    # Test connectivity
  top_n: 200                  # Keep only top 200 by latency
  max_ping_ms: 500           # Discard servers > 500ms

output:
  output_dir: "output"        # Output directory
  write_base64: true          # Generate base64 file
  write_csv: true             # Generate CSV report

See full configuration reference for all options.

Fetching Configurations

Fetch configurations from sources without testing:

# Fetch all sources
massconfigmerger fetch

# Fetch from last 12 hours only
massconfigmerger fetch --hours 12

# Fetch specific protocols
massconfigmerger fetch --fetch-protocols vmess,vless

# Save to custom directory
massconfigmerger fetch --output-dir /path/to/output

What Happens During Fetch?

  • Reads URLs from sources.txt
  • Downloads configurations from each source
  • Parses and validates configurations
  • Removes duplicates
  • Saves to output/vpn_subscription_raw.txt

Merging & Testing

Test connectivity and merge configurations:

# Test and merge all configs
massconfigmerger merge

# Use existing file (skip fetching)
massconfigmerger merge --resume output/vpn_subscription_raw.txt

# Keep only top 100 by latency
massconfigmerger merge --top-n 100

# Filter by protocols
massconfigmerger merge --include-protocols VMESS,VLESS

# Exclude slow servers
massconfigmerger merge --max-ping 300
Note: Testing thousands of servers can take 10-20 minutes. Use --top-n to limit the number of configs.

Output Files

File Format Use Case
vpn_subscription_raw.txt Plain text Raw configuration links
vpn_subscription_base64.txt Base64 Import via URL in most VPN clients
clash.yaml YAML Clash, Clash Meta, Clash for Windows
vpn_singbox.json JSON Sing-Box, Stash
vpn_detailed.csv CSV Analysis in Excel/spreadsheets
vpn_report.json JSON Detailed statistics and metadata

Advanced Usage

Geographic Filtering

# Include only specific countries
massconfigmerger merge --include-country US,GB,JP,DE

# Exclude specific countries
massconfigmerger merge --exclude-country RU,CN

Pattern Matching

# Include configs matching pattern
massconfigmerger merge --include-pattern "premium|vip|pro"

# Exclude configs matching pattern
massconfigmerger merge --exclude-pattern "free|trial|test"

Multiple Output Formats

# Generate Surge config
massconfigmerger merge --output-surge output/surge.conf

# Generate Quantumult X config
massconfigmerger merge --output-qx output/quantumultx.conf

Automation

Using Cron (Linux/macOS)

# Edit crontab
crontab -e

# Add this line to run every 6 hours
0 */6 * * * cd /path/to/project && massconfigmerger full

Using Docker Compose

# Create docker-compose.yml
version: '3.8'
services:
  merger:
    image: massconfigmerger/massconfigmerger:latest
    volumes:
      - ./output:/app/output
    environment:
      - MERGE_INTERVAL=21600  # 6 hours
    command: |
      sh -c "while true; do
        massconfigmerger full
        sleep $MERGE_INTERVAL
      done"

# Start the service
docker-compose up -d

GitHub Actions

# .github/workflows/update.yml
name: Update Subscriptions
on:
  schedule:
    - cron: '0 */6 * * *'  # Every 6 hours
  workflow_dispatch:

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      - name: Install
        run: pip install massconfigmerger
      - name: Run
        run: massconfigmerger full
      - name: Commit
        run: |
          git config user.name "Bot"
          git config user.email "bot@example.com"
          git add output/
          git commit -m "Update subscriptions"
          git push

Troubleshooting

Command not found

Problem: massconfigmerger: command not found

Solution:

# Ensure pip installed to user directory
pip install --user massconfigmerger

# Add to PATH (add to ~/.bashrc or ~/.zshrc)
export PATH="$HOME/.local/bin:$PATH"

Timeout Errors

Problem: Frequent timeout errors during fetching

Solution:

# Increase timeout in config.yaml
network:
  request_timeout: 30
  concurrent_limit: 10  # Reduce concurrent requests

High Memory Usage

Problem: Process uses too much memory

Solution:

# Limit configs and enable periodic saves
massconfigmerger merge --top-n 1000 --save-every 100
